## Ask mode

- Include diffs for proposed edits.

**Example:**

```diff
- old line
+ new line
```

---

## Environment

- Use `bun` as runtime, test runner, and package manager.

**Examples:**

```bash
bun i
bun <script>
bunx <tool>
```

---

## Communication

- Assume good intent.
- Aim for high skimmability.
- Only explain what isn’t trivial.
- Have strong opinions, loosely held.
- Seek the collective truth, not just one opinion.

## Git

- Use the `git-commit` skill to commit.
- After commit, run `git pull` to sync local with origin.
