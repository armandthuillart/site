# My Stack

<p className="text-muted-foreground">December 2025 - Armand Thuillart</p>

Here’s where I’m currently most productive:

## 1. Frontend (TanStack and React)

I’ve started with [Next.js](https://nextjs.org/) because of its popularity, automatic integration with [Vercel](https://vercel.com/home), and solid developer experience.

As time passed, I also played a bit with [Nuxt.js](https://nuxt.com/) for fun but turned out I wasn’t a big fan of [Vue.js](https://vuejs.org/). 

I then landed on [Remix](https://remix.run) and pretty much liked it to be honest. However, what’s ahead for the latter is [uncertain](https://remix.run/blog/wake-up-remix).

Until one day, I realized that I didn’t need all the RSC and SSR features Next.js provides. My apps are either static with some dynamic parts, or heavily client-side rendered.

Here comes [TanStack Start](https://tanstack.com/start/latest), it keeps the best parts of Remix and Next.js, while giving me full control over the architecture.

## 2. Styling (Tailwind CSS and shadcn/ui)

Building flexible, accessible components is difficult. You either use (and extend) a component library, or build your own. This is why I now use [shadcn/ui](https://ui.shadcn.com/).

It provides well-designed and extensible components, built on top of accessible, unstyled primitives like [Base UI](https://base-ui.com/). This includes basics like buttons and inputs, but also icons, charts, and even custom themes.

Components are styled with [Tailwind CSS](https://tailwindcss.com/), the most AI-friendly CSS library. Why? This makes generating and editing code with AI tools much easier.

Tailwind uses a compiler to generate only the classes used. So while the utility CSS framework contains many possible class names, only the classes used will be included in the single, compiled CSS file.

Assuming you only write Tailwind code, your bundle will never be larger than the total set of used Tailwind classes. It’s extremely unlikely you would use them all. This means you have a fixed upper bound on the size of the generated CSS file, which is then minified, compressed, and cached for the best performance.

## 3. Backend (Convex)

Convex is the best, you type something and it just works. They have production-ready [components](https://convex.dev/components) like crons, workflow, and many other addons that are exhausting features to develop or maintain.

The best part? Convex is entirely typed, which is heaven to for a [TypeScript](https://www.typescriptlang.org/) lover like me.

## 4. AI (Cursor, OpenCode, and Warp)

[Cursor](https://cursor.com/home) is my everyday code editor. I can ask questions, plan features, debug code, and get crazy-good code completions.

Thanks to [MCP](https://modelcontextprotocol.io) ([Context7](https://context7.com/)). Cursor Agent has up-to-date knowledge of Convex, React, TanStack so it’s almost accurate all time.

[OpenCode](https://opencode.ai) is great, pair it with [GitHub Copilot](https://github.com/features/copilot) subscription, helps me vibecoding features without breaking anything.

[Warp](https://warp.dev) is my everyday terminal. This is great to do a lot of things system-wide, like managing packages, running scripts, or even using AI to help me with commands.

AI tools aren’t perfect yet, but they’re a huge time saver.
